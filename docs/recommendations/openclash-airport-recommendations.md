## 先说结论

OpenClash 场景里，最重要的通常不是节点数量，而是：

- 订阅兼容性
- 教程清不清楚
- 默认组和规则好不好承接
- 高峰期稳不稳

所以更适合先看“能不能顺利接进去”，再看“名字排第几”。

## OpenClash 场景最该看什么

### 1. 订阅兼容性

这是第一位。  
如果导入都不顺，后面再多节点也没有意义。

### 2. 默认规则和默认组好不好用

新手更怕的不是不会优化，而是一开始就：

- 组太乱
- 规则太乱
- 根本不知道先测什么

### 3. 路由器教程是否清楚

有没有：

- 导入说明
- 代理组说明
- 常见问题
- DNS 和规则基础说明

### 4. 多设备高峰期表现

全屋场景下，问题会被明显放大，所以更要看高峰期表现。

## 更省时间的顺序

1. 先跑通手机或电脑
2. 再导入 OpenClash
3. 先用默认规则和默认组
4. 最后再测全屋设备
