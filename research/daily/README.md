# 每日记录

这个目录用来存放每天的研究快照。

## 文件命名

- `YYYY-MM-DD.md`

例如：

- `2026-06-07.md`
- `2026-06-08.md`

## 每日记录最少包含什么

- 当天检查时间
- 检查了哪些来源
- 哪些候选有变化
- 哪些变化只进内部层
- 哪些变化建议进公开层
- 下一步要跟进什么

## 建议节奏

- 没变化也可以留一条“今日无实质变化”
- 有变化时要明确写：是官网变了、候选状态变了，还是第三方页更新了

## 自动生成方式

- 本地可运行：`scripts/create-daily-check.sh`
- GitHub Actions 会在每天北京时间 `09:30` 自动生成当天文件
- 自动任务只创建骨架，具体变化和判断仍然需要人工补充
- 周末可再把本周有价值的变化整理进 `research/weekly/`
